2️⃣ ADDED FILE-BASED DATA PERSISTENCE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

NEW FILE: data/users.txt
Format: Plain text, one account per line
Example:
  john_doe,password123
  sarah_smith,wellness_2024
  admin,password123

How It Works:
1. App Start:
   LoginApplication() → loadUserDatabase() → reads data/users.txt into HashMap
   
2. First Run:
   data/users.txt doesn't exist yet (will be created)
   Only admin/password123 available
   
3. User Sign-Up:
   User enters credentials → validation checks → saveUserDatabase()
   → new user written to data/users.txt
   
4. App Restart:
   LoadUserDatabase() reads all previously saved users
   All previous accounts available for login
   
5. Data Survives:
   Even after app closes → data/ folder remains
   Users don't need to re-register
   Full account history preserved

Methods Added:
- loadUserDatabase(): Reads data/users.txt and populates HashMap
- saveUserDatabase(): Writes current HashMap to data/users.txt
